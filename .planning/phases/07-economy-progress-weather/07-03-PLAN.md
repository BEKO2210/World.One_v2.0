  <action>
Create detail/topics/space.js implementing the full topic contract. This is the most interactive module in Phase 7 -- ISS position updates every 10 seconds with orbit trail visualization.

**Imports:** i18n, DOMUtils, MathUtils, fetchTopicData, fetchWithTimeout, createTierBadge, ensureChartJs, createChart, CHART_COLORS, toRgba

**Meta:** { id: 'space', titleKey: 'detail.space.title', category: 'progress', icon: '' }

**Module state:** let _intervals = [], _trail = [], _chartData = null
const MAX_TRAIL = 30 (approximately 5 minutes of positions at 10s intervals)

**Continent paths:** Copy the CONTINENT_PATHS array from earthquakes.js (simplified continent outlines as SVG path d-strings). This is the same world map base used for earthquakes and conflicts.

**ISS SVG Map (block 0 -- primary-chart, rendered directly in render()):**
- Build SVG with DOMUtils.createSVG: viewBox="0 0 900 450", width="100%", style with dark background and border-radius
- Add continent path outlines (rgba(255,255,255,0.05) fill, 0.15 stroke)
- Add latitude/longitude grid lines (same pattern as earthquakes.js)
- Add orbit trail polyline: fill="none", stroke="rgba(0,255,204,0.4)", stroke-width="1.5", stroke-dasharray="4,2"
- Add ISS dot circle: r="6", fill="rgba(0,255,204,0.9)", stroke="rgba(0,255,204,0.3)", stroke-width="3"
- Store references to trail polyline and dot elements

**ISS position polling (started in render()):**
- Primary API: https://api.wheretheiss.at/v1/satellites/25544 (fetchWithTimeout 8000ms)
- Update function:
  1. Fetch ISS position (latitude, longitude)
  2. Convert to SVG coords: MathUtils.geoToSVG(lat, lng, 900, 450)
  3. Push {x, y} to _trail array; if length > MAX_TRAIL, shift oldest
  4. **Date line detection:** if consecutive points have |dx| > 400, clear _trail (prevents cross-map line artifact)
  5. Update trail polyline points attribute
  6. Update dot cx/cy attributes
  7. Update position text display (lat/lng formatted)
- Call update() immediately for first position
- _intervals.push(setInterval(update, 10000)) for 10s refresh
- On fetch error: console.warn('[Space] ISS fetch failed'), keep last known position, show static badge

**Crew list (block 1 -- trend area):**
- Hardcode current ISS Expedition crew (changes every ~6 months, not worth live API):
  Crew members with name, nationality, role (commander/flight engineer), agency (NASA/Roscosmos/ESA/JAXA)
  Approximately 7 crew members (Expedition 74, early 2026)
- Build as a styled list with DOMUtils.create: flag emoji by nationality, name, role badge

**Spaceflight news feed (block 3 -- tiles area):**
- Fetch via fetchTopicData('space') to get space-news.json cache
- Display up to 6 articles from articles array
- Each article: title (as link), news_site, published_at (formatted date), summary (truncated to ~100 chars)
- Build with DOMUtils.create as card grid

**Satellite count visualization (block 4 -- explanation area):**
- Hardcode: ~13,000 active satellites (UCS Satellite Database 2025)
- Show breakdown by operator: SpaceX/Starlink ~6,000, OneWeb ~600, Planet Labs ~200, others ~6,200
- Simple horizontal bar representation using DOMUtils.create (DOM bars, not Chart.js)

**Context tiles:**
- Tile 1: ISS altitude (~408 km)
- Tile 2: ISS speed (~27,600 km/h)
- Tile 3: Crew members count
- Tile 4: Orbits per day (~16)

**getChartConfigs():** Return empty array -- all visualizations are DOM/SVG-based, no Chart.js needed for this topic

**cleanup():** Clear ALL intervals (_intervals.forEach(clearInterval), _intervals = []), clear _trail = [], _chartData = null. This is CRITICAL -- ISS polling must stop on navigation.

Create detail/topics/weather.js implementing the full topic contract. Fetches live weather data for 24 cities from Open-Meteo and renders SVG sparklines for each.

**Imports:** i18n, DOMUtils, MathUtils, fetchWithTimeout, createTierBadge

**Meta:** { id: 'weather', titleKey: 'detail.weather.title', category: 'realtime', icon: '' }

**Module state:** let _chartData = null (no intervals needed -- one-shot fetch)

**24 cities array (module constant WEATHER_CITIES):**
Copy the exact city list from 07-RESEARCH.md (24 cities with name, lat, lng across Europe 5, North America 4, South America 3, Asia 5, Africa 4, Oceania 2, Antarctica 1).

**WMO weather code mapping (module constant WMO_CODES):**
Copy the WMO_CODES object from 07-RESEARCH.md with desc, icon (text label, not actual icon file), and severity (0-4).
Add severityColor function: severity >= 4 -> '#ff3b30' (red), >= 3 -> '#ff9500' (orange), >= 2 -> '#ffcc00' (yellow), else 'var(--text-secondary)'.

**Weather data fetch (in render()):**
- Build URL per city: `https://api.open-meteo.com/v1/forecast?latitude=${lat}&longitude=${lng}&hourly=temperature_2m,weather_code&forecast_hours=24&timezone=auto`
- Use Promise.allSettled to fetch all 24 cities in parallel (not Promise.all -- individual failures must not break others)
- For each settled result:
  - fulfilled: extract hourly.temperature_2m (24 values) and hourly.weather_code (24 values)
  - rejected: mark city as unavailable, show "--" for temperature
- Display cities progressively as a grid of cards

**City card rendering:**
For each city, build a card with DOMUtils.create containing:
1. City name (bold)
2. Current temperature (first hourly value) with degree symbol
3. Current weather description from WMO_CODES[currentCode].desc
4. SVG sparkline of 24 hourly temperatures:
   - Use DOMUtils.createSVG('svg', { viewBox: '0 0 120 40', width: '120', height: '40' })
   - Create polyline from hourly temps: map each temp to x/y coordinates
   - x = (i / 23) * 120, y = 40 - ((temp - min) / (max - min || 1)) * 36 - 2
   - Polyline: fill="none", stroke with CHART_COLORS.realtime color, stroke-width="1.5"
5. Warning badge if max weather_code in 24h exceeds thresholds:
   - code >= 95: thunderstorm warning (red)
   - code >= 75: heavy snow warning (orange)
   - code >= 65: heavy rain warning (orange)
   - Use severityColor() for badge background

**Card grid layout:**
- CSS grid via style attribute: 4 columns on wide screens (grid-template-columns: repeat(auto-fill, minmax(200px, 1fr)))
- Each card: glass-morphism style (rgba background, border-radius, padding)
- Cards with warnings get a colored left border matching severity

**Hero block (block 0):**
- Show summary: "24 cities worldwide" as hero count
- createTierBadge with 'live' tier (data fetched live from Open-Meteo)
- Show extremes after data loads: hottest city and coldest city with temperatures

**Extreme weather warnings section (block 1 -- trend area):**
- After all cities loaded, collect all cities with weather_code exceeding thresholds
- Display as a warning list: city name, warning type, severity color badge
- If no extreme weather: show "No severe weather warnings" message

**Context tiles (block 3):**
- Tile 1: Hottest city right now (name + temp)
- Tile 2: Coldest city right now (name + temp)
- Tile 3: Cities with warnings (count)

**Explanation, Comparison, Sources (blocks 4-6):** Text from i18n keys. Source: Open-Meteo (free weather API)
