  <action>
Create js/utils/data-loader.js as a new standalone ES6 module (NOT extending the existing js/data-loader.js).

Export two named functions:

1. `fetchWithTimeout(url, timeoutMs = 5000)` -- Fetches a URL with AbortSignal.timeout(). Returns the Response object. Throws on timeout or network error.

2. `fetchTopicData(topic, apiUrl = null)` -- The main 3-tier fallback function:
   - Tier 1 (live): If apiUrl is provided, call fetchWithTimeout(apiUrl, 5000). On success, parse JSON, return `{ data, tier: 'live', age: 0 }`. Log success with `console.log('[DataLoader] Live data for ${topic}')`.
   - Tier 2 (cache): Fetch `data/cache/${topic}.json`. On success, parse JSON, calculate age from `_meta.fetched_at` timestamp if present (age in milliseconds), return `{ data, tier: 'cache', age }`. If no `_meta.fetched_at`, age = null.
   - Tier 3 (static): Fetch `data/fallback/static-values.json`, extract the topic key, return `{ data: topicData, tier: 'static', age: null }`. If topic key not found or value is null, return `{ data: null, tier: 'static', age: null }`.

   Each tier is wrapped in try/catch. On failure, log `console.warn('[DataLoader] {tier} failed for ${topic}:', err.message)` and fall through to next tier.

Use a module-level cache for static-values.json so it is fetched only once: `let _staticCache = null;`. The helper `_getStaticFallback(topic)` loads and caches the full JSON, then returns the topic key's value.

Follow established conventions:
- Named exports (no default export)
- Private helpers prefixed with underscore: `_getStaticFallback`
- Console logging with `[DataLoader]` prefix
- 2-space indentation, single quotes for strings
- Decorative comment separators matching existing style

Do NOT use localStorage for topic data caching (that is the existing DataLoader's pattern). This module uses file-based cache via fetch().
