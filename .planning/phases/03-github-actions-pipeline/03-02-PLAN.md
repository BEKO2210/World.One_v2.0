All three scripts follow the established pattern: `#!/usr/bin/env node` shebang, ESM imports from `./cache-utils.js`, async `main()`, header/summary logging.

**cache-economy-ext.js** (ACTIONS-05):
- **currencies.json**: Fetch `https://open.er-api.com/v6/latest/USD` via `fetchJSON()`. Extract `rates` object. Filter to major pairs: EUR, GBP, JPY, CNY, INR, BRL, RUB, KRW, CHF, AUD, CAD, MXN. Save as `{ base: 'USD', rates: { EUR: x, GBP: x, ... }, rate_date: response.time_last_update_utc }`
- **inequality.json**: Fetch World Bank Gini index `SI.POV.GINI` for world: `https://api.worldbank.org/v2/country/WLD/indicator/SI.POV.GINI?format=json&per_page=60&date=1960:2025`. Use `extractWorldBankEntries()`. Note: World Bank Gini for "WLD" may return sparse data. If fewer than 3 data points, also try top-10 countries individually: `USA;CHN;IND;BRA;DEU;GBR;ZAF;NGA;RUS;JPN` with `per_page=5&mrnev=1` (most recent non-empty value). Save as `{ world_trend: [...], country_latest: [{ country, code, gini, year }] }`
- **poverty.json**: Fetch World Bank poverty headcount `SI.POV.DDAY` ($2.15/day) for world: `https://api.worldbank.org/v2/country/WLD/indicator/SI.POV.DDAY?format=json&per_page=60&date=1990:2025`. Use `extractWorldBankEntries()`. Save as `{ poverty_trend: [...] }`
- `saveCache()` for each -- 3 calls

**cache-progress-ext.js** (ACTIONS-06):
- **arxiv-ai.json**: Fetch arXiv API `http://export.arxiv.org/api/query?search_query=cat:cs.AI+OR+cat:cs.LG&sortBy=submittedDate&sortOrder=descending&max_results=20` via `fetchText()`. Parse the Atom XML response using regex (matching existing collect-data.js approach, NOT xml2js which is a heavy dep):
  - Extract `<entry>` blocks using regex `/<entry>([\s\S]*?)<\/entry>/g`
  - From each entry extract: `<title>`, `<published>`, `<summary>` (first 200 chars), `<link href="..."/>` (the abstract URL)
  - Build array of `{ title, published, summary, url }`
  - Save as `{ papers: [...], total_results: N }` where total_results comes from `<opensearch:totalResults>`
- **space-news.json**: Fetch `https://api.spaceflightnewsapi.net/v4/articles/?limit=10&ordering=-published_at` via `fetchJSON()`. Extract `results` array. Map to `{ title, url, image_url, news_site, published_at, summary }`. Save as `{ articles: [...] }`
- `saveCache()` for each -- 2 calls

**cache-disasters.js** (ACTIONS-07):
- **disasters.json**: Fetch ReliefWeb API `https://api.reliefweb.int/v1/disasters?appname=worldone&limit=20&sort[]=date:desc&fields[include][]=name&fields[include][]=date&fields[include][]=type&fields[include][]=country&fields[include][]=status` via `fetchJSON()`. The response has `data` array with `fields` objects. Map to `{ name, date, type: fields.type[0]?.name, countries: fields.country?.map(c=>c.name), status }`. Save as `{ disasters: [...] }`
- **hunger.json**: Fetch World Bank undernourishment `SN.ITK.DEFC.ZS` for world: `https://api.worldbank.org/v2/country/WLD/indicator/SN.ITK.DEFC.ZS?format=json&per_page=60&date=2000:2025`. Use `extractWorldBankEntries()`. Save as `{ undernourishment_trend: [...] }`
- `saveCache()` for each -- 2 calls

Error handling for all: wrap each API call in try/catch, log warnings for failures, continue to next file. Only `process.exit(1)` if ALL files in a script fail (zero files written).

  <name>Task 2: Create validate-cache.js validation utility</name>
  <files>scripts/validate-cache.js</files>
  <action>
Create `scripts/validate-cache.js` as the central validation script referenced in 03-VALIDATION.md.

**Basic mode** (`node scripts/validate-cache.js`):
- Read all `.json` files in `data/cache/` (excluding meta.json)
- For each file, verify:
  1. Valid JSON (parseable by JSON.parse)
  2. Has `_meta` object at top level
  3. Has `_meta.fetched_at` string that parses as a valid Date
  4. Has at least one data key besides `_meta`
- Print results as a table: filename, valid (yes/no), age (time since fetched_at), data keys count
- Exit 0 if all files pass, exit 1 if any fail
- If no cache files exist, print warning and exit 0 (not an error -- files may not be generated yet)

**Full mode** (`node scripts/validate-cache.js --full`):
- Everything in basic mode, PLUS:
- Check expected file list: `['biodiversity.json', 'co2-history.json', 'ocean.json', 'solar.json', 'population.json', 'freedom.json', 'conflicts.json', 'currencies.json', 'inequality.json', 'poverty.json', 'arxiv-ai.json', 'space-news.json', 'disasters.json', 'hunger.json']`
- Report which expected files are missing
- Exit 1 if any expected files are missing

**Workflow check mode** (`node scripts/validate-cache.js --check-workflow`):
- Read `.github/workflows/cache-pipeline.yml`
- Verify it contains all 6 job names: `update-biodiversity`, `update-environment-ext`, `update-society-ext`, `update-economy-ext`, `update-progress-ext`, `update-disasters`
- Verify each job has `timeout-minutes: 10` and `continue-on-error: true`
- Exit 0 if all checks pass, exit 1 with details if any fail

Use ESM imports. Parse `process.argv` for flags. Log with `[validate-cache]` prefix per project convention.
