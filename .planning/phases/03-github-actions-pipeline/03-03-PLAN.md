**1. Create `.github/workflows/cache-pipeline.yml`:**

Structure: Single workflow with 6 cron triggers and `github.event.schedule`-based conditional execution per job.

```yaml
name: Cache Pipeline — Detail Page Data

on:
  schedule:
    - cron: '0 3 * * *'     # 03:00 UTC - biodiversity
    - cron: '30 3 * * *'    # 03:30 UTC - environment-ext
    - cron: '0 4 * * *'     # 04:00 UTC - society-ext
    - cron: '30 4 * * *'    # 04:30 UTC - economy-ext
    - cron: '0 5 * * *'     # 05:00 UTC - progress-ext
    - cron: '30 5 * * *'    # 05:30 UTC - disasters
  workflow_dispatch:
    inputs:
      job_group:
        description: 'Which cache group to update (or "all")'
        required: false
        default: 'all'
        type: choice
        options:
          - all
          - biodiversity
          - environment-ext
          - society-ext
          - economy-ext
          - progress-ext
          - disasters

permissions:
  contents: write

concurrency:
  group: cache-pipeline-${{ github.event.schedule || 'manual' }}
  cancel-in-progress: false   # Never cancel a running cache job

env:
  NODE_VERSION: '20'
```

**6 jobs**, each following this template:

```yaml
  update-{GROUP}:
    if: >-
      github.event.schedule == '{CRON}' ||
      (github.event_name == 'workflow_dispatch' && (github.event.inputs.job_group == 'all' || github.event.inputs.job_group == '{GROUP}'))
    runs-on: ubuntu-latest
    timeout-minutes: 10
    continue-on-error: true
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'

      - name: Install dependencies
        run: npm ci || npm install

      - name: Run cache script
        env:
          UCDP_API_TOKEN: ${{ secrets.UCDP_API_TOKEN }}  # Only in society-ext job
        run: node scripts/cache-{GROUP}.js

      - name: Update meta.json
        if: always()
        run: |
          node scripts/generate-meta.js --job {GROUP} --status ${{ job.status }}

      - name: Commit and push cache files
        if: always()
        run: |
          git config user.name "World.One Pipeline"
          git config user.email "pipeline@world.one"
          git add data/cache/*.json
          if git diff --staged --quiet; then
            echo "No changes to commit"
          else
            git commit -m "chore(cache): update {GROUP} cache $(date -u '+%Y-%m-%d %H:%M UTC')"
            git push || {
              git pull --rebase origin $(git branch --show-current) && git push
            }
          fi
```

IMPORTANT: The "Update meta.json" step MUST have `if: always()` so that job failures from the cache script step are still recorded in meta.json. Without this guard, a failing cache script (exit 1) causes GitHub Actions to skip subsequent steps by default, preventing jobs_failed from being counted. The "Commit and push" step also needs `if: always()` so that even a partial meta.json update gets committed.

Job-to-cron-to-script mapping:
| Job | Cron | Script | Env |
|-----|------|--------|-----|
| update-biodiversity | '0 3 * * *' | cache-biodiversity.js | (none) |
| update-environment-ext | '30 3 * * *' | cache-environment-ext.js | (none) |
| update-society-ext | '0 4 * * *' | cache-society-ext.js | UCDP_API_TOKEN |
| update-economy-ext | '30 4 * * *' | cache-economy-ext.js | (none) |
| update-progress-ext | '0 5 * * *' | cache-progress-ext.js | (none) |
| update-disasters | '30 5 * * *' | cache-disasters.js | (none) |

CRITICAL details:
- `UCDP_API_TOKEN` env var ONLY on the `update-society-ext` job (the only one that needs it)
- `continue-on-error: true` at JOB level (not step level) -- a failing API should not block git commit of partial results
- `if: always()` on both "Update meta.json" and "Commit and push" steps -- ensures failure tracking and commit happen even when the cache script step fails
- `cancel-in-progress: false` -- never cancel a running cache job because another schedule triggered
- Concurrency group includes `${{ github.event.schedule || 'manual' }}` so different schedule triggers don't conflict
- Each job does its own checkout, setup, npm install -- jobs run on separate runners and share NO state

**2. Create `scripts/generate-meta.js`:**

ESM script that reads current cache state and updates `data/cache/meta.json`.

Arguments: `--job {name}` (which job just ran), `--status {success|failure}` (job outcome)

Logic:
1. Read existing `data/cache/meta.json` if it exists (initialize empty object if not)
2. List all `.json` files in `data/cache/` excluding `meta.json`
3. For each file, read it and count data points:
   - If top-level contains an array -> length
   - If top-level contains objects with arrays -> sum of array lengths
   - Simple heuristic: count top-level keys minus `_meta`
4. Update `job_details[jobName]` with `{ status, updated_at: ISO, files: [filenames] }`
5. Recount `jobs_ok` and `jobs_failed` from `job_details` statuses
6. Set `total_cache_files` = count of JSON files (excluding meta.json)
7. Set `total_data_points` = sum of all data point counts
8. Set `last_full_update` = current ISO timestamp
9. Write `data/cache/meta.json` with `JSON.stringify(meta, null, 2)`

Output schema matching ACTIONS-09:
```json
{
  "last_full_update": "2026-03-21T05:30:00.000Z",
  "jobs_ok": 5,
  "jobs_failed": 1,
  "total_cache_files": 14,
  "total_data_points": 847,
  "job_details": {
    "biodiversity": { "status": "ok", "updated_at": "...", "files": ["biodiversity.json"] },
    "environment-ext": { "status": "ok", "updated_at": "...", "files": ["co2-history.json", "ocean.json", "solar.json"] }
  }
}
```
