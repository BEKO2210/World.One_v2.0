Fix the root cause: IntersectionObserver never observes any section element because register() is called before init().

**Primary fix in js/scroll-engine.js init() method (line 32-40):**
After the call to `this._setupObservers()` (which creates `this._observer`), add a retroactive observation loop that observes all sections already registered via earlier register() calls:

```javascript
init() {
  this._setupObservers();
  this._setupRevealObserver();
  this._setupScrollListener();
  this._updateMetrics();

  // Retroactively observe sections registered before init()
  if (this._observer) {
    for (const [, section] of this.sections) {
      this._observer.observe(section.element);
    }
  }

  this.running = true;
  this._tick();
  return this;
}
```

This is the correct fix location (scroll-engine.js, not app.js) because:
- The engine's API should support both call orders (register-before-init AND register-after-init)
- Swapping the call order in app.js alone would fix the current bug but leave scroll-engine.js fragile for future callers
- The existing `if (this._observer)` guard in register() already handles the register-after-init case correctly

**Do NOT change the call order in app.js** -- the engine should be resilient to either order. The register-before-init pattern is actually a valid use case (collect all sections, then start the engine).
