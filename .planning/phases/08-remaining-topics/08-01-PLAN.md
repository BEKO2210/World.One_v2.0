   - KP_BANDS array mapping Kp values to aurora visibility latitudes and colors:
     Kp 9 = 30deg lat (rgba(255,0,100,0.15)), Kp 7 = 40deg (rgba(255,100,0,0.12)), Kp 5 = 50deg (rgba(0,255,100,0.10)), Kp 3 = 60deg (rgba(0,100,255,0.08))
   - Continent SVG paths reused from earthquakes.js pattern (simplified outlines, 900x450 viewport)

4. **render(blocks):**
   - **Hero block:** Fetch cache via `fetchTopicData('solar')` for predicted cycle data. Then fetch LIVE observed sunspot count from `https://services.swpc.noaa.gov/json/solar-cycle/observed-solar-cycle-indices.json` via fetchWithTimeout(url, 8000). Use the most recent entry's `ssn` field as hero value. Show createTierBadge. Display current sunspot number and cycle phase text.
   - **Chart block:** Reserved for solar cycle 25 chart (lazy via getChartConfigs). Create canvas with id `solar-cycle-chart`.
   - **Trend block:** Fetch live Kp index from `https://services.swpc.noaa.gov/json/planetary_k_index_1m.json` via fetchWithTimeout(url, 8000). Parse JSON array, take last entry's `kp_index` value. Build SVG aurora Kp-index map (900x450 viewport):
     - Draw continent outlines (same simplified paths from earthquakes.js/space.js)
     - For each KP_BAND where kp <= currentKp: draw horizontal latitude band using MathUtils.geoToSVG(band.lat, 0, 900, 450) for yNorth and MathUtils.geoToSVG(-band.lat, 0, 900, 450) for ySouth. Bands are filled rectangles spanning full width with the band's color.
     - Add legend showing current Kp value and aurora visibility latitude
   - **Tiles:** Solar wind speed (from cache if available, else static "400 km/s"), cycle phase ("Maximum" for 2024-2026 Cycle 25 peak), next minimum estimate (~2030)
   - **Explanation block:** i18n text about solar cycles, 11-year pattern, aurora connection
   - **Comparison block:** Compare current Cycle 25 to historical peak (Cycle 19: SSN 285 in 1958)
   - **Sources:** NOAA SWPC, SILSO (Royal Observatory of Belgium)

5. **getChartConfigs():** Return one chart config for solar cycle 25:
   - Parse `observed-solar-cycle-indices.json` data (stored in module state from render): filter entries from 2019 onward (Cycle 25 start). Labels = "time-tag" (YYYY-MM), data = "ssn" values.
   - Overlay predicted data from cache solar.json: labels = "time-tag", data = "predicted_ssn" with confidence bands (high_ssn / low_ssn as fill between).
   - Chart type: 'line'. Observed = solid realtime color. Predicted = dashed line with lighter fill for confidence envelope.
   - Uses dual datasets: observed (solid) and predicted (dashed with `borderDash: [5, 5]`), plus two hidden fill datasets for high/low bounds.

6. **cleanup():** Null module state variables (_observedData, _predictedData, _currentKp).
